## Credit-Based Usage

Each API request costs **credits** based on the capabilities used. This ensures you pay proportionally to the compute resources consumed.

| Capability | Credits | Description |
|:-----------|:--------|:------------|
| Chat | 1 | Conversational AI |
| Data | 1 | Structured data processing |
| Files | 2 | File upload/download via signed URLs |
| Research | 3 | Web search + synthesis |
| Code | 3 | Sandboxed code execution |
| Documents | 5 | PDF/report generation |

A request using multiple capabilities costs the **sum** of their weights. For example, a request with `capabilities: ["research", "documents"]` costs 8 credits.

## Rate Limits

All plans are rate-limited to **60 requests per minute** at the middleware level.

If you exceed this, you'll receive a `rate_limited` error with HTTP status 429. Wait and retry with exponential backoff.

```json
{
  "error": {
    "code": "rate_limited",
    "message": "Too many requests. Limit: 60/minute."
  }
}
```

## Usage Tracking

Every response includes current usage in the `usage` object:

```json
{
  "usage": {
    "credits_used": 42,
    "credits_limit": 500,
    "credits_this_request": 3,
    "overage": false,
    "tokens_used": 847
  }
}
```

| Field | Description |
|:------|:------------|
| `credits_used` | Credits consumed this billing period |
| `credits_limit` | Your plan's included monthly credits |
| `credits_this_request` | Credits consumed for this specific request |
| `overage` | Whether you've exceeded your included credits |
| `tokens_used` | LLM tokens consumed for this specific request |

## Rate Limit Headers

Every response includes rate limit headers:

| Header | Description |
|:-------|:------------|
| `X-RateLimit-Limit` | Monthly credit limit for your plan |
| `X-RateLimit-Remaining` | Credits remaining this billing period |
| `X-RateLimit-Reset` | Unix timestamp when the limit resets |

<Note>
  Free plan has a hard cap at 50 credits. Paid plans allow overages — requests are never blocked, and overage charges are billed at the end of each billing period.
</Note>
